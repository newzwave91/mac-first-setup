# 맥 처음 세팅 도우미 — 설계 문서

- 작성일: 2026-07-19
- 상태: 설계 확정 대기 (사용자 리뷰 전)
- 배포 대상: 본인 + 지인 소규모
- 형태: **하이브리드** — 터미널 접점은 "붙여넣기 1회", 이후 모든 상호작용은 macOS 네이티브 팝업창

## 1. 목적

맥을 처음 산 완전 입문자가 명령어 한 줄만 붙여넣으면, 이후에는 익숙한 macOS 팝업창을 클릭하는 것만으로 (1) 추천 앱 설치, (2) 기본 시스템 설정, (3) 남은 수동 작업 안내까지 받는 도구.

앱/설정 추천 목록은 한국어 유튜브·블로그·커뮤니티 리서치에 근거한 큐레이션 문서에서 확정하며, 출처를 남겨 이후 갱신 기준으로 삼는다.

## 2. 형태 결정의 근거 (대안 검토 기록)

- **미서명 .app**: macOS Sequoia 이후 우클릭-열기 우회가 제거되어 "시스템 설정 → 그래도 열기" 관문 필요 — 입문자에게 터미널보다 어려움. 기각.
- **서명·공증 .app**: Apple Developer Program $99/년 + 공증 파이프라인 유지 필요 — 소규모 배포에 과함. 보류 (이후 확장 옵션).
- **하이브리드 (채택)**: `curl` 한 줄은 Gatekeeper를 아예 거치지 않고, 실행 직후부터 osascript 네이티브 다이얼로그로 진행 — 비용 0, 입문자 체감은 "명령어가 앱을 띄워줌".

## 3. 구성요소

### 3.1 `setup.sh` — 단일 파일 오케스트레이터

- **셸 호환성**: macOS 기본 `/bin/bash`(3.2) 호환으로 작성. 외부 의존성 0 (뭔가 설치되기 전에 동작해야 함).
- **배포 실행 형태**: `bash -c "$(curl -fsSL <raw URL>)"` (실제 URL은 GitHub 공개 리포 생성 후 확정) — 파이프(`curl | bash`)가 아닌 이 형태로 안내하여 stdin/TTY를 유지한다 (Homebrew 설치기 등 하위 프로세스의 대화형 입력 대비).
- **UI 레이어**: osascript(AppleScript) 래퍼 함수 모음
  - `display dialog` — 환영/확인/완료 안내
  - `choose from list ... with multiple selections allowed` — 프로필 선택(단일), 앱/설정 선택(복수)
  - `display notification` — 단계 진행 알림
  - `display alert` — 오류 안내
  - 사용자가 [취소]를 누르면 어느 단계에서든 안전 종료 (이미 완료된 항목은 유지, 리포트 출력)
- **관리자 권한**: 가능한 한 `osascript ... with administrator privileges`의 GUI 암호 팝업을 사용. Homebrew 설치기가 터미널 암호 입력을 요구하는 경우에 대비해, 웹 안내 페이지에 "화면에 표시되지 않지만 입력되고 있음" 그림 안내 포함.
- **데이터 블록**: 파일 상단에 앱 카탈로그를 구분자 레코드로 내장
  - 필드: `id | 카테고리 | 설치방식(cask/mas) | 패키지명 | 표시명 | 한 줄 설명 | 기본선택 프로필들`
  - 목록 수정 = 이 블록 수정 (별도 빌드 없음)
- **실행 단계**:
  1. 환영 팝업 — 무엇을 하는지/하지 않는지 설명, 동의 후 진행
  2. 프로필 선택: 사무·일반 / 학생·문서작업 / 크리에이터 / 개발자
  3. 앱 선택 팝업 — 프로필 기본 체크셋이 미리 선택된 상태에서 가감
  4. 시스템 설정 선택 팝업 — 추천값 기본 선택 (트랙패드 탭 클릭·세 손가락 드래그, 키 반복속도, Finder 확장자 표시, 스크린샷 저장 폴더, Dock 정리, 원화(₩)→백틱 등 한국 사용자 특화 포함)
  5. 실행: Xcode Command Line Tools → Homebrew → 선택한 cask/mas 설치 → `defaults` 적용 → `dockutil` Dock 구성
  6. 완료 리포트: 성공/실패/수동 할 일(앱 권한 허용, App Store·계정 로그인 등)을 팝업 + `~/맥세팅-리포트.txt`로 저장

### 3.2 안내 웹페이지 (`guide/`)

- GitHub Pages로 같은 리포에서 호스팅
- 내용: ① 터미널 여는 법(Cmd+Space → "터미널", 그림), ② 큰 [복사하기] 버튼, ③ 붙여넣고 return, ④ 이후 팝업 흐름 미리보기, ⑤ 암호 입력 안내
- 순수 정적 HTML 1장 (외부 의존성 없음)

### 3.3 리서치 산출물 (`docs/research/app-curation.md`)

- 구현 전 deep-research로 한국어 유튜브·블로그·커뮤니티(클리앙 등) 조사
- 프로필 4종 × 카테고리(브라우저/메신저/생산성/유틸리티/미디어/개발)별 추천 앱 + 추천 시스템 설정, **각 항목에 출처 명시**
- 이 문서가 카탈로그 데이터 블록의 단일 근거(SSOT). 목록 갱신은 문서 갱신 → 데이터 블록 반영 순서

## 4. 안전 설계

- 파괴적 작업(삭제·덮어쓰기) 일절 없음
- `defaults` 변경 전 현재값을 백업하고 `~/맥세팅-복구-YYYYMMDD.sh` 복구 스크립트 자동 생성 (실행 시 원복)
- 항목별 오류 처리: 한 항목 실패 시 중단하지 않고 계속, 리포트에 실패 목록 명시 (`set -e` 전역 사용 금지)
- **멱등성**: 재실행 안전 — 설치된 앱은 스킵, 적용된 설정은 재적용해도 무해
- `--dry-run` 플래그: 실제 변경 없이 수행 예정 목록만 출력 (개발 검증 겸용)

## 5. 지원 범위

- Apple Silicon + 최근 macOS (Sonoma~Tahoe). Intel 맥은 미보장 명시
- macOS 버전별 `defaults` 키 차이는 버전 감지 후 분기, 미지원 버전이면 해당 항목 스킵 + 리포트 기재

## 6. 검증 계획

- `shellcheck` 통과
- `--dry-run` 전수 확인 (선택 조합별)
- 본인 맥에서 실제 실행 → 리포트·복구 스크립트 동작 확인
- 새 사용자 계정(또는 게스트 계정)에서 실행해 신품 유사 환경 검증
- 지인 1명 파일럿 후 배포

## 7. 범위 밖 (YAGNI)

- 서명된 .app 패키징, 자동 업데이트, 텔레메트리, Intel 지원, MDM/조직 배포
- 웹 생성기(체크박스로 맞춤 스크립트 생성)는 이후 확장 후보로만 기록
